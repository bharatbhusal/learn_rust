Modules

Modules:

  1. Definition:
    • A module is a way to organize code into separate namespaces and scopes in Rust.
    • Modules help in organizing and structuring your codebase, making it more manageable and reusable.
    • Modules facilitate encapsulation and help avoid naming conflicts.
  1. Creating a Module:
    • You can create a module by using the mod keyword followed by the module's name and a block that contains its contents.
    • The module's name should be a valid Rust identifier and is typically written in lowercase.
    • Here's an example of creating a module named my_module:
      mod my_module {
    // Module contents go here
}
  1. Module Hierarchy:
    • Modules can be nested to create a hierarchical structure.
    • This allows you to organize related functionality and create sub-modules within a module.
    • Here's an example of creating a nested module structure:
      mod my_module {
    mod sub_module {
        // Sub-module contents go here
    }
}
  1. Visibility:
    • By default, the items (functions, structs, enums, etc.) inside a module are private to that module and not accessible from outside.
    • You can use the pub keyword to make items public and accessible from other modules.
    • Public items can be accessed using the :: syntax, specifying the module path followed by the item's name.
    • Here's an example of making a struct public within a module:
      mod my_module {
    pub struct MyStruct {
        // Struct fields go here
    }
}

// Accessing the public struct from outside the module
let my_instance = my_module::MyStruct { /* Initialize fields */ };
  1. Using Module Contents:
    • To use items (functions, structs, etc.) defined within a module, you can either fully qualify their names or use the use statement to bring them into scope.
    • The use statement allows you to access items without the need for fully qualifying their names.
    • Here's an example of using the use statement to bring a function into scope:
      mod my_module {
    pub fn my_function() {
        // Function body goes here
    }
}

// Bringing the function into scope
use my_module::my_function;

// Using the function
my_function();
  1. File-based Modules:
    • Rust has a convention of organizing code into separate files that correspond to modules.
    • Each file acts as a module, and its contents can be accessed from other modules.
    • You can use the mod keyword followed by the file name (without the .rs extension) to reference the module.
    • For example, a file named my_module.rs can be referenced as mod my_module; within another module.

Modules are a powerful tool for organizing and structuring your codebase. They allow you to encapsulate functionality, control visibility, and create a hierarchy of namespaces. By utilizing modules effectively, you can create more maintainable and reusable code in Rust.
