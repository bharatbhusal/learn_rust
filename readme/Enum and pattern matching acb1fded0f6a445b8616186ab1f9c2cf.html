Enum and pattern matching

Enums:

  1. Definition:
    • An enum (short for enumeration) is a custom data type that allows you to define a type by enumerating its possible variants.
    • Enums are used to represent a fixed set of values or states.
  1. Declaration:
    • You can declare an enum using the enum keyword followed by the enum's name and a list of variants.
    • Each variant can optionally have associated data.
    • Here's an example of an enum declaration:
      enum TrafficLight {
    Red,
    Yellow,
    Green,
}
  1. Instantiation:
    • To create an instance of an enum variant, you can use the enum's name followed by the variant's name.
    • Here's an example of instantiating a TrafficLight enum:
      let light = TrafficLight::Red;
  1. Associated Data:
    • Enum variants can have associated data, allowing you to attach additional information to each variant.
    • Here's an example of an enum with associated data:
      enum Message {
    Quit,
    Move { x: i32, y: i32 },
    Write(String),
    ChangeColor(i32, i32, i32),
}
    • You can provide the associated data when instantiating the enum variants.
  1. Match Expressions:
    • Pattern matching in Rust allows you to match values against patterns and execute different code based on the matched pattern.
    • The match expression is used for pattern matching.
    • Here's an example of using match to handle different enum variants:
      match light {
    TrafficLight::Red => println!("Stop!"),
    TrafficLight::Yellow => println!("Prepare to stop!"),
    TrafficLight::Green => println!("Go!"),
}

Pattern Matching:

  1. Matching Enum Variants:
    • With pattern matching, you can match specific enum variants and execute code based on the matched variant.
    • You can also extract associated data from the matched variant.
    • Here's an example of matching an enum variant with associated data:
      match message {
    Message::Quit => println!("Received quit message"),
    Message::Move { x, y } => println!("Move to ({}, {})", x, y),
    Message::Write(text) => println!("Write: {}", text),
    Message::ChangeColor(r, g, b) => println!("Change color to RGB({}, {}, {})", r, g, b),
}
  1. Matching with Wildcards:
    • You can use the _ wildcard pattern to match any value without binding it to a variable.
    • It is useful when you want to ignore certain cases or provide a default behavior.
    • Here's an example of using the _ wildcard pattern:
      match value {
    Some(data) => println!("Got data: {:?}", data),
    None => println!("No data"),
    _ => unreachable!(), // Catch-all case
}
  1. Matching with Guards:
    • Guards allow you to add additional conditions to patterns using the if keyword.
    • You can use guards to add more complex conditions to the patterns.
    • Here's an example of using a guard in a pattern:
      match value {
    Some(data) if data > 0 => println!("Positive value: {}", data),
    Some(data) if data < 0 => println!("Negative value: {}", data),
		Some(_) => println!("Zero"),
		None => println!("No data"),
}

Enums and pattern matching are powerful features in Rust that enable you to work with different states or variants of data and handle them in a structured way. They provide a concise and safe approach to handle complex scenarios, making your code more expressive and robust.
